# 1) 링커? 라이브러리?
최근에서야 C++을 배워서 컴파일 한다고 설쳐서 링커라는 존재를 알게 되었다.  
빌드를 위해서는 헤더 파일을 포함시키는 것뿐만 아니라, 라이브러리 파일이 있는 경로를 지정해서 라이브러리 파일을 가리켜야 그 안에 있는 특정 기능이나 작업을 수행하는 함수, 클래스, 변수 같은 것들을 불러올 수 있다.  

## 1> 정적 라이브러리
확장자는 '.lib', '.a'를 가진다.  
컴파일될 때 라이브러리 코드가 실행 파일에 포함된다.  
용량이 무거워지는 단점은 있지만, 따로 파일을 챙기지 않아도 실행 파일이 실행되기 때문에 배포가 간단해진다.  

## 2> 동적 라이브러리
확장자는 '.dll'(윈도우), '.so'(리눅스), '.dylib'(MacOS)을 가진다.  
동적 라이브러리는 실행 파일과 별도로 존재하고, 프로그램 실행 시에 불러오는 작업을 통하여 사용된다.  
실행 파일의 크기를 줄이고 여러 프로그램들이 같은 라이브러리를 사용하는 경우 메모리 사용을 최적화할 수 있다.  
하지만, 실행 파일의 정상 동작을 위해서는 동적 라이브러리 파일도 반드시 들고 있어야 실행되는 불편함이 있다.

# 2) opencv 라이브러리
```bash
find /opt/fsl-imx-xwayland -name "libopencv*"
```  
🔼 내가 사용하는 우분투 내에 opencv 라이브러리 파일을 찾기 위해 위의 명령어를 실행했다.  

<br>

아래는 위의 명령어로 찾은 라이브러리들이 각각 어떤 기능을 위해 필요한지 적은 내용이다.  
- libopencv_core : opencv의 핵심 기능 포함. 기본 데이터 구조, 행렬 연산, 메모리 관리 등의 기본적인 기능
- libopencv_imgproc : 이미지 처리에 관련된 기능. 이미지 필터링, 기하학적 이미지 변환, 컬러 공간 변환, 히스토그램 등을 포함
- libopencv_highgui : 이미지와 비디오 관련 인터페이스 제공. 윈도우 관리, 키보드/마우스 이벤트 처리, 이미지와 비디오의 읽기, 쓰기 등이 포함
- libopencv_ml : 기계 학습 모듈. 분류, 회귀, 클러스터링, 기타 머신 러닝 알고리즘 제공
- libopencv_video : 비디오 분석과 관련된 기능 제공. 모션 추적, 배경 감산 등 포함
- libopencv_objdetect : 객체 탐지 기능 제공. 얼굴 인식, 객체 탐지 등의 기능 포함
- libopencv_dnn : 딥 러닝 기능을 지원. 다양한 딥 러닝 프레임워크로 학습된 모델을 로드, 실행 가능
- libopencv_imgcodecs : 이미지 파일의 읽기와 쓰기를 위한 기능 제공. 다양한 형식의 이미지 파일 사용 가능
- libopencv_videoio : 비디오 입출력 관련 기능을 제공. 비디오 파일을 읽고 쓰거나 카메라로부터 스트리밍하는 기능을 제공
