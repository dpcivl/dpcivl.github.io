# 0) 개요
## 1> 문제 상황
11월 27일부터 해서 오늘까지.. imx8mplus 개발 보드에서 VX delegate를 이용해서 NPU에서만 객체 인식을 추론하려고 시도했었는데, 첫번째로는 적당한 모델을 찾지 못해서 시간을 소모했고, 두번째로는 모델은 찾았으나 프로그램을 개발하지 못해서 소모하고 있다. (중간에 다른 일도 하긴 했다..)  

imx8mplus 개발보드 상에서 객체 인식을 위해서는 추천하는 모델을 사용해 추론해야 할 것 같은데 그 모델이 mobilenet-ssd 모델과 yolov4 모델이다.  
이 모델들을 인터넷에서 찾아서 어떻게든 tflite 모델로 만들어보려고 시도해봤으나, tflite 모델로 변환 시 모델 성능이 아예 안 나오는 현상을 발견했고, 그 후로 다시 자료를 찾다가 eIQ Toolkit UserGuide에서 추천하는 객체 인식 모델을 eIQ 디렉토리 내에서 제공한다는 사실을 최근에서야 알게 됐다.  

변환 또한 eIQ Toolkit에서 가능하지만, 변환한 tflite 모델을 가지고 추론을 해야 하는데 내가 만들려고 하는 프로그램과 비슷한 예제 코드가 잘 없어서 여기저기 찾아다니면서 프로그램을 만들어보려고 한다.  

## 2> 맨땅에 헤딩
그렇기 때문에 이번 글은 원래부터 도움이 안되는 내 블로그지만 더더욱 도움이 안될 것이다.  
이번 글에서는 내가 이번 문제를 해결하는 과정을 담을 생각이다.  
**정보를 얻고자 한다면 뒤로가기를 누르는 게 나을 수도 있겠다.**  

# 1) 입력 데이터 코드 작성
우선은 여태 삽질하면서 내가 완전히 이해한 부분만 기존 코드에서 복붙했다.  
NMS 기법 적용이나, 이미지 전처리, 추론 알고리즘 같은 부분은 복붙해서 사용하긴 했었지만 정확히 알지는 못하기 때문에 일단 보류하고,  
라이브러리를 불러온다든지, 실시간 객체 인식을 위해 카메라 캡처를 생성한다든지 하는 기본적인 동작들의 코드만 우선 작성했다.  

이후에 모르는 부분은 공부하면서 작성했다.  
아래는 공부한 내용들이다.   

## 1> 인터프리터 준비
- `allocate_tensors()` : tflite 인터프리터에 메모리를 할당한다. 
- `get_input_details()` : 모델에 어떠한 입력을 넣어줘야 하는지에 대한 정보를 담고 있다. 
- `get_output_details()` : 모델이 어떤 출력을 내보내는지에 대한 정보를 담고 있다. 

## 2> BGR2RGB
OpenCV는 기본적으로 이미지를 BGR 형식으로 처리한다고 한다.  
그러나 대부분의 이미지 처리 알고리즘은 RGB를 사용한다.  
그렇기 때문에 OpenCV의 캡처를 사용할 때도, 입력받는 `frame`에 대하여 BGR 형식을 RGB 형식으로 바꿔주는 작업이 필요하다.  

## 3> 이미지 전처리
Tensorflow Lite 모델에 입력하기 전에 이미지 데이터를 입력 텐서 형식에 알맞게 변환하는 것이 필요하다.  
다음은 구체적으로 어떠한 과정을 거쳐서 준비하는지 적었다.  

### 1] 이미지의 데이터 타입 변환
현재 내가 사용하는 모델은 입출력 모두 float32 형태의 데이터를 받기로 되어있다.  
그런데 cv2에서 불러오는 이미지는 일반적으로 uint8 타입의 데이터이다.  
그러므로 모델이 요구하는 float32 형태의 데이터로 변환해주는 작업이 필요하다.  

### 2] 정규화
uint8 타입의 데이터를 float32 타입의 데이터로 바꾸어주면 데이터 형식은 바뀌지만, 그 값은 변하지 않는다.  
그러므로 해당 데이터를 255로 나눠줘서 zero-centered를 진행하고, 이후에 정규화까지 처리해준다.  

### 3] 가상의 배치 차원 추가
현재 내가 사용하는 모델의 shape는 [1, 300, 300, 3]이다.  
그런데 카메라 캡처에서 가져오는 이미지는 [높이, 너비, 채널]의 차원으로 구성된다.  
그렇기 때문에 가상의 배치 차원(크기가 1인 차원)을 추가해준다.  
해당 과정은 이미지 전처리를 다 실행하고 난 후 실시되어야 한다.  

## 4> 인터프리터에 데이터 설정
### 1] `set_tensor`
`set_tensor`를 이용해서 앞의 과정에서 전처리한 데이터를 가져와서 입력 텐서에 넣는다.  
입력 텐서의 shape과 이미지 데이터의 shape이 같다면, 에러 없이 정상적으로 설정된다.  

### 2] `invoke`
`invoke` 메서드를 실행하면, 인터프리터가 입력 텐서에 들어오는 데이터를 가지고 추론하게 한다.  

# 2) 출력 데이터 코드 작성
드디어 출력 코드다.  
데이터를 출력하는 과정에서는 출력 텐서의 데이터 값도 제대로 받아와야 하고,  
받아온 데이터 값으로 바운딩 박스랑 스코어도 표출해야 하고,  
바운딩 박스가 겹치지 않도록 NMS 기법도 사용해야 하고,  
라벨맵에서 클래스를 가져와서 해당 객체가 어떤 객체인지 올바르게 추론하고 있는지도 확인해야 한다.  

## 1> 출력 데이터 제대로 갖다쓰기
모델마다 출력 텐서의 개수와 인덱스가 달라서 이 부분에서 되게 삽질을 많이 했다.  
내가 현재 사용하는 모델은 출력 텐서가 2개이다.  
- 바운딩 박스 좌표 : [ymin, xmin, ymax, xmax] 형식으로 된 좌표이다.  
- 클래스 신뢰도 점수 : 검출된 객체에 대해서 각각의 클래스가 어느 정도의 스코어를 가지고 있는지 출력한다.  

### 1] 첫번째 출력 - [1, N, 1, 4]
먼저, 첫번째 출력인 바운딩 박스 좌표의 값을 가져오기 위해서, 차원 축소를 실행했다.  
1의 값을 가지는 첫번째와 3번째 차원을 없애고 [N,4]의 형태로 바꿔서 감지된 객체의 바운딩 박스를 가져오도록 하였다.  

### 2] 두번째 출력 - [1, N, 91]
두번째 출력은 클래스와 스코어의 정보를 담고 있다.
