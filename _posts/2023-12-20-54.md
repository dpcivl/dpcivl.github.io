# 1) 프로젝트 구조 계획

- 소스 파일과 헤더 파일 분리 : C++에서는 보통 '.cpp' 파일에 구현하고, '.h' 파일에 선언한다.  

- 디렉토리 구조 : 소스 파일과 헤더 파일을 다른 디렉토리에서 관리한다. 



<br>



# 2) 빌드 시스템 선택

- Makefile

- CMake : 'CMakeLists.txt' 파일을 통해 프로젝트를 구성하고, 다양한 컴파일러와 환경에서 사용 가능하다. 



<br>



# 3) 기본적인 프로젝트 파일

- main.cpp : 메인 소스 파일

- 기능별 모듈 파일 : 프로젝트의 각 기능을 별도의 클래스나 함수로 구현한다. 



<br>



# 4) 라이브러리 관리

외부 라이브러리를 프로젝트에 포함시키기 위해서 'Makeflie'이나 'CMake'에서 라이브러리 경로와 링커 설정을 해줄 수 있다. 



<br>



# 5) 버전 관리 시스템 사용

**Git**과 같은 버전 관리 시스템을 사용하여 소스 코드의 변경사항을 추적한다. 



<br>



# 6) 빌드 및 테스트

- 빌드 : 'CMake'를 사용하는 경우, 'cmake' 명령어로 빌드 파일을 생성하고, 'make'를 사용하여 프로젝트를 빌드한다. 

- 테스트 : 테스트 케이스를 작성하고 실행한다. 



<br>



# +) 간단한 C++ 프로젝트 구조

```makefile
MyProject/
│
├── CMakeLists.txt  # CMake 설정 파일
│
├── include/        # 헤더 파일 디렉토리
│   └── myclass.hpp
│
├── src/            # 소스 파일 디렉토리
│   ├── main.cpp
│   └── myclass.cpp
│
└── docs/           # 문서 디렉토리
```
